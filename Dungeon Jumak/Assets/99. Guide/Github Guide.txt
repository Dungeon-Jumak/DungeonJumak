원활한 협업을 통해 깃허브 가이드 내에 있는 프로세스 대로 협업을 진행한다.
다음과 같은 프로세스로 진행하지 않을 경우 태형에 처한다.

[브랜치]
브랜치는 상황에 따라 다음과 같이 나누며, 각각의 기능은 다음과 같다.

main (Master) : 출시될 수 있는 브랜치 (버전 관리만 담당)
hotfix : 출시 버전에서 발생한 버그를 수정하는 브랜치
develop : 다음 출시 버전을 준비하는 브랜치 (계속 유지)
feature : 기능 개발 브랜치 (기능 제작 후 삭제)
release : 출시 버전을 준비하는 브랜치

[기본 작업 브랜치 개설 방법]
- develop -> feature(기능의 이름에 따라 브랜치 이름이 달라짐)  -> 개인 브랜치 (개인 이름)

[협업 프로세스]

1. 깃허브 내에서 Fork를 한 후 로컬에 clone을 받는다. (Fork한 레포를 받는 것임)

2. 깃허브 내 Issues 탭에서 각자 작업한 것과 작업할 것을 작성한다. (중복 작업 방지, 필수)
	+ 템플릿이 존재하므로 템플릿에 맞게 작성하면 된다.

3. 각자 개발후 자신의 원격 저장소로 push한다.

4. 팀장 develop 브랜치로 풀리퀘를 보낸다. (Fork 레포 -> 팀장 레포)

5. 팀장 코드리뷰 후 머지

* 풀리퀘스트 사항이 머지되기 전에 팀원이 수정해서 푸쉬할 경우 자동으로 풀리퀘에 계속해서 반영된다.

6. 머지된 후 모든 개발자는 pull을 받는다. (저장소 최신화)
